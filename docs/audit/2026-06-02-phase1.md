# BPM Badminton — Code Audit, Phase 1 (Targeted, Read-Only)

_Generated 2026-06-02. **Report only — no app code was changed.** Findings are for triage._

## How this was produced

A multi-agent fan-out reviewed the highest-leverage surfaces (security/auth, the ~133 `catch` blocks, Cosmos correctness, the top hotspot files, and perf) against the project's own rules in `CLAUDE.md`. **Every high-severity finding was then adversarially re-verified by a separate agent that tried to refute it** — all 8 survived (`real=true`). Two complementary passes ran alongside: a tooling check (ESLint/deps) and a live Playwright UX pass. Medium/low findings were not individually re-verified — treat them as high-quality leads, confidence noted per row.

## Executive summary

**57 findings** — 9 high, 18 medium, 30 low.


By dimension: silent-bug 26, code-quality 9, logic 8, security 8, efficiency 6.


### Top themes

1. **The "lying empty state" footgun recurs (the project's #1 historical risk).** A failed Cosmos/fetch is swallowed and rendered as legitimate "no data" in **at least 4 verified spots** — `GET /api/players`, `GET /api/skills`, `PaymentsCard.loadPlayers`, `ProfileTab` — exactly the pattern that caused the v1.3 outage. This is the highest-priority cluster.

2. **Admin-auth contract is under-enforced.** Nearly every *mutating* admin route uses the cheap sync `isAdminAuthed` (signature+expiry only) instead of `isAdminAuthedWithMember` (re-checks `role==='admin'`), despite `lib/auth.ts`'s own docs saying to use the async one for mutations — so a demoted admin keeps write powers for up to the 30-day cookie TTL. Plus two **unauthenticated write paths** (`equipment/gear` PUT, `games` POST `sessionId` override) and a **name-only first-PIN claim** on `members/me`.

3. **Lint is dead.** The Next 16 / ESLint 10 bumps silently broke the entire lint toolchain, and CI never ran it — a whole class of issues is uncaught.

4. **A few real correctness/data bugs** beyond silent failures: the Cover-action shows for already-paid players (money logic), SetupPage collapses multi-purchase bird usages on save (data loss), and `/api/sessions/recent` does an N+1 query.
