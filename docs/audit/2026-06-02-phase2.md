# BPM Badminton — Code Audit, Phase 2 (Full Sweep, Read-Only)

_Generated 2026-06-02. **Report only — no app code was changed.** Covers the ~150 files not deep-read in Phase 1._

## How this was produced

Eight batch reviewers swept the remaining surface (auth sheets, the SPA consumer shell, Stats components, the rest of the Command Center, admin misc + hooks, lib modules, the money-math routes, and the remaining REST routes) against the `CLAUDE.md` rubric. High-severity findings were adversarially re-verified — **5 of 6 confirmed real, 1 refuted and excluded** (listed at the bottom). Medium/low are single-pass leads.

## Executive summary

**56 findings** — 5 high, 18 medium, 33 low.


By dimension: silent-bug 22, logic 20, design-ux 4, a11y 4, code-quality 4, security 1, type-design 1.


### Phase 2 themes

1. **The lying-empty-state cluster is systemic.** Phase 2 adds **3 more** confirmed spots — `PlayersTab`, `BirdInventoryCard`, `AnnouncementsCard` — to the 4 from Phase 1, for **7 total**. Several don't even call `reportFetchFailure()`, so the app-wide offline banner stays silent. This is one fix *pattern* applied 7 times, not 7 unrelated bugs.

2. **The `PUT /api/session` doc-wipe is reachable from a SECOND editor.** Phase 1 found it via `SessionDetailsEditor`; Phase 2 independently re-found it via `DateTimeEditor` (date/time save) — same root cause (`app/api/session/route.ts` rebuilds the doc from a fixed key-set without reading existing), confirming any admin session edit can silently drop `settled`/`approvedNames`/`prev*`. This raises its priority.

3. **Error-handling inconsistency across the auth sheets.** `EnterCodeSheet` is the one sheet missing a `catch` and a `>=500` split that its three siblings have — so a server 5xx reads as 'wrong code' and burns the rate limit. Small, mechanical fixes.

4. **A11y gaps** on the primary path: the name autocomplete has no combobox ARIA roles; a couple of hand-rolled dialogs lack focus traps (the repo's own `BottomSheet` primitive solves this).
