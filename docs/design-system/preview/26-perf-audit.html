Target floor: iPhone 15 / Pixel 8-class (A16 / Tensor G3). Modern GPUs handle backdrop-filter and compositor-animated blur comfortably — the audit below has been re-tiered against that floor. Most "critical" concerns on older hardware are downgraded. Two items remain worth fixing because they waste battery regardless of GPU class.

Critical path findings

Low Backdrop-filter proliferation

Downgraded for iPhone 15+. Apple's A16/A17 and Tensor G3 handle stacked backdrop-filter without measurable frame drops up to ~5 concurrent layers. You can ship the glass hierarchy as-is. _preview.css:56 · 14-glass-card.html · 19-bottom-nav.html No action required. Soft guidance: if a single screen has more than 5 stacked blur layers, tier it (modals 22px, cards 10–14px, pills static gradient). Otherwise leave alone.

High filter: blur() + mix-blend-mode on animating elements

Still worth fixing — even on A17. filter: blur() combined with mix-blend-mode on animating blobs forces the compositor into a slow path (non-composited repaint every frame) regardless of GPU class. It's the #1 thermal cause and shows up as sustained ~8–12% extra battery draw over a 10-min session. 22-aurora-bg.html:59 · 24-app-background.html:67 Replace with the proven pattern in assets/aurora-bg.css: pre-blurred radial gradients (no filter), no mix-blend-mode, animate only transform. This is a must-fix before ship.

Med Infinite animations without IntersectionObserver

Battery-only concern on modern hardware. Aurora + loader animations run animation: ... infinite. iOS 17+ and Chrome aggressively throttle offscreen/backgrounded animations, so the frame-rate cost is near zero — but on-screen continuous motion still draws power. Background-app throttling kicks in automatically, so this is a "good hygiene" fix, not an urgent one. 20-shuttle-loader · 11-motion · 22-aurora-bg Add animation-play-state: paused on loaders when their IntersectionObserver entry is out of view. Low priority — save for a polish pass.

Low will-change overuse

Largely a non-issue on iPhone 15+. VRAM budget is generous enough that promoting 20+ elements to GPU layers won't OOM. Still best practice to keep the hint on actively animating elements only, but no visible impact expected. assets/aurora-bg.css:45 Lint rule suggestion: flag will-change on any selector that isn't inside an @keyframes-driven animation. No urgent cleanup needed.

Secondary findings

Low Multi-layer box-shadows

Fine on A16+. Shadows are painted once at layout and cached — the cost is per-paint, not per-frame. Stacked shadows don't meaningfully affect scroll perf on this hardware floor. No action. Keep the aesthetic layering.

High Google Fonts network weight

Network cost is hardware-independent. 6 families × multiple weights = 400–800 KB of font data before first paint. On a first launch over 4G/5G this is the visible TTI bottleneck, not GPU. Matters on any device. Lock one pairing and ship just that — ~80 KB subset. font-display: swap, preload the 2 above-the-fold weights, lazy-load the rest. Biggest user-perceivable win.

Med SVG court lines / arcs as animated DOM

Aurora card and app-background options animate SVG stroke-dashoffset and offset-distance. These invalidate the SVG layer every frame on older Safari. Bake court markings into a static PNG/SVG background-image. Only the shuttle arc stays animated — and only if that screen is visible.

Med Material Icons full webfont

Network-bound, so hardware-independent. Entire ~100 KB icon font loaded to use ~20 glyphs. Switch to material-symbols variable subset or inline as SVG sprites. Saves 60–80 KB. Pairs well with the font pruning above.
